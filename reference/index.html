Design System // w-b.dev

Visual Language

The shared design tokens, typography, components, and animation patterns used across all w-b.dev hotsites. Dark monochrome. Helvetica + JetBrains Mono. One accent color. Scroll-driven.

Reference implementation: pipeline-deep-dive.w-b.dev

01 // Color Tokens

Palette

A dark monochrome base with a single warm accent. Hierarchy is built through opacity, not hue variation. The foreground color #f3f1ec is warm paper — never pure white.

Opacity Ladder

The core design principle: never introduce a new hue. Use the opacity ladder to create depth.

02 // Typography

Two Faces, One Voice

Helvetica for display and narrative. JetBrains Mono for data, labels, code, and metadata. No other typefaces. Ever.

Display / H1
Helvetica 700 · clamp(42px, 6vw, 72px) · -0.04em · line-height 1.0
Section Heading / H2
Helvetica 700 · clamp(24px, 3.5vw, 40px) · -0.03em · line-height 1.1
Subsection / H3
Helvetica 700 · clamp(18px, 2.5vw, 28px) · -0.02em · line-height 1.2
Body text uses the dim color for comfortable long-form reading against the dark background. Bold spans promote to full foreground for emphasis.
Helvetica 400 · clamp(16px, 1.6vw, 19px) · color: --dim · max-width 680px · line-height 1.65
Section Label // Overline
JetBrains Mono 400 · 11px · 0.18em tracking · uppercase · color: --dim
Mono body — data values, descriptions, metadata fields
JetBrains Mono 400 · 13px · 0.02em tracking · color: --dim
Tertiary Meta Label
JetBrains Mono 400 · 10px · 0.16em tracking · uppercase · color: --dimmer

Rules

PrincipleDetail
Negative tracking for displayH1: -0.04em, H2: -0.03em, H3: -0.02em. Tighter = larger.
Positive tracking for labelsMono labels: +0.08em to +0.28em. Wider = smaller.
Clamp everythingAll display sizes use clamp() for fluid responsiveness. No breakpoints for type.
body max-width: 680pxLong-form paragraphs never exceed 680px. Readability over width.
03 // Spacing & Layout

Generous Emptiness

Whitespace is a feature, not waste. Sections breathe. Content clusters are separated by 50–60px. Page sections use 120px vertical margin.

05 // Animation

Scroll-Driven, Never Gratuitous

All animation is triggered by IntersectionObserver. No libraries. Elements start hidden (opacity:0; transform:translateY(30px)) and reveal when their section enters the viewport.

Breathe
Pulse Scale
Fade Slide In

Patterns

PatternCSSWhen
Section revealopacity 0.7-0.8s + translateY(30-40px)Every section entering viewport
Child staggertransition-delay: nth-child * 0.1-0.15sCards, steps, list items
Breatheopacity 0.3→1 over 2s, infiniteSubtle ambient pulse on idle elements
Draw-in (SVG)stroke-dashoffset 1000→0 over 1.2sDiagram edges appearing
Scale-in (nodes)scale(0.5)→scale(1) + opacity over 0.6sCircle nodes popping in
/* Scroll-driven reveal — the only JS needed */ const observer = new IntersectionObserver((entries) => { entries.forEach(entry => { if (entry.isIntersecting) entry.target.classList.add('visible'); }); }, { threshold: 0.15, rootMargin: '0px 0px -10% 0px' }); document.querySelectorAll('.section') .forEach(s => observer.observe(s));
06 // Page Structure

Anatomy of a Page

Every w-b.dev hotsite follows this skeleton. Sections are full-viewport height on desktop, fluid on mobile. Each section has a numbered label, a divider, and a heading before its content.

Hero — title, subtitle, visual anchor
1
Section — label "01 // name" + divider + h2 + body + component
2

N
More sections as needed...
E
