02 // Normalization & Enrichment

Making Raw Data Machine-Readable and Context-Rich

Raw indicators arrive in dozens of incompatible formats — STIX 2.1, CSV, JSON, YARA, plain text. A normalization layer maps everything to a unified schema before enrichment workers layer on geolocation, ASN ownership, passive DNS history, WHOIS records, sandbox detonation results, and MITRE ATT&CK technique mappings. Only enriched records advance to the correlation engine.

Format Parsing

Schema Normalization

STIX 2.1, OpenIOC, MISP JSON, CSV, and free-text are all parsed into a canonical indicator record with fixed fields: type, value, confidence, tlp, first_seen, last_seen, source_id.

Geolocation

IP Context Enrichment

MaxMind GeoIP2 and IPinfo supplement every IP indicator with country, city, ASN, and organization. Hosting provider fingerprints flag bulletproof hosters known to tolerate abuse.

Passive DNS

Domain History Resolution

pDNS lookups reconstruct historical A, AAAA, MX, and NS records. Infrastructure pivots reveal shared hosting patterns that connect seemingly unrelated attacker campaigns.

ATT&CK Mapping

TTP Classification

Behavioral indicators are mapped to MITRE ATT&CK tactics and techniques. Every enriched record carries a tactic vector — enabling tactic-aware prioritization downstream.

Sandbox Detonation

Dynamic Malware Analysis

Suspicious files and URLs are submitted to an isolated detonation cluster. Behavioral reports — process trees, network connections, registry mutations — are attached to the indicator record.

Confidence Scoring

Reliability Weighting

A weighted Bayesian model scores each indicator 0–100 based on source reputation, corroboration across feeds, age, and detonation confirmation. Low-confidence records are quarantined for analyst review.

03 // Detection Rules

Converting Intel into Enforced Policy

Enriched indicators feed a rule-generation engine that produces Sigma rules, YARA signatures, and SIEM correlation logic automatically for high-confidence records. Human analysts review medium-confidence candidates before promotion. Every rule carries provenance metadata linking it back to its source indicators — so when an indicator ages out, affected rules are flagged for reassessment automatically.
